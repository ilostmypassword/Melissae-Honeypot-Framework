# Contributing to Melissae

**Discord**: [Join the server](https://discord.gg/RXWn85cnYm) — the best place to discuss ideas before opening a PR.

---

## Ways to Contribute

- New honeypot or CVE modules
- Dashboard improvements (React/Tailwind)
- Scoring signals
- Documentation fixes
- Bug reports

## Roadmap

| Area | Task |
|------|------|
| Modules | Improve MQTT log parsing |
| Modules | New CVE modules |
| Modules | SNMP honeypot |
| Intelligence | MITRE ATT&CK mapping |
| Reporting | Automated PDF/HTML reports |
| Analysis | Behavioral clustering |
| Deception | Adaptive honeypot responses |
| Dashboard | Rules UI for scoring thresholds |

---

## Pull Request Process

1. Fork and create a branch (`feat/my-module`)
2. Test end-to-end: install, enroll an agent, enable the module, check the dashboard
3. Open a PR against `v2.1` with a short description, the protocol/port used, and a sample log entry

For large changes, open an issue first to align on the approach.
